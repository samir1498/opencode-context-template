# Research Repo

Investigation output. You asked a question, got an answer, saved it.

## Convention

```
personal-research/
├── engineering-and-product/
│   ├── 2026-06-17-topic-name.md
│   └── another-finding.md
├── marketing-and-content/
│   └── competitor-analysis.md
├── seo-and-growth/
│   └── 2026-06-03-blog-keyword-status.md
└── README.md
```

## Rules

- One topic per file. Self-contained — the agent reads one file and has all the context.
- No deep nesting. Two levels max (domain/filename.md).
- Timestamp prefix when chronology matters (`YYYY-MM-DD-slug.md`).
- Domain folders = categories that make sense for your work.
- Agent writes findings back to the relevant file, never creates new files without asking.

## Research vs Context

- **Research** = investigation output. Saved answers.
- **Context** = live planning. What you're doing right now.

Keep them separate.
